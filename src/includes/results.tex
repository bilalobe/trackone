\subsection{OpenTimestamps Integration}
\textbf{Successful Bitcoin anchoring:} Production day blobs anchored to Bitcoin blocks 919384, 919388, 919418 via OpenTimestamps calendar servers.

\textbf{Verification validated:}
\begin{itemize}[nosep]
    \item Local verification with Bitcoin Core (headers-only mode)
    \item Multiple independent calendar attestations received
    \item Merkle root recomputation matches anchored commitment
    \item End-to-end trustless timestamp verification confirmed
\end{itemize}

\textbf{CI challenges addressed:}
\begin{itemize}[nosep]
    \item Full blockchain sync impractical ($>$500 GB, days to sync)
    \item Headers-only mode reduces storage to $<$1 GB
    \item Initial header sync: $\sim$15 minutes (one-time cost)
    \item Subsequent verifications: seconds (with cached headers)
    \item Alternative: offline verification with pre-downloaded headers
\end{itemize}

\textbf{Operational considerations:}
\begin{itemize}[nosep]
    \item \texttt{ots stamp}: Immediate (free, public calendars)
    \item \texttt{ots upgrade}: Automatic after 1--6 hours (Bitcoin confirmation)
    \item \texttt{ots verify}: Requires Bitcoin P2P or block explorer API
    \item Air-gapped deployments: Defer verification or use trusted header feeds
\end{itemize}
