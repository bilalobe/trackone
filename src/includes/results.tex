\subsection{OpenTimestamps Integration}

\textbf{Production Bitcoin anchoring validated:}
\begin{itemize}[nosep]
    \item Day blobs successfully anchored to Bitcoin blocks 919384, 919388, 919418
    \item Multiple calendar server attestations received (a.pool.opentimestamps.org, etc.)
    \item Local verification with Bitcoin Core (headers-only mode, pruned-safe)
    \item Merkle root recomputation matches anchored commitment (end-to-end consistency)
\end{itemize}

\textbf{OTS workflow timing characteristics:}
\begin{itemize}[nosep]
    \item \texttt{ots stamp}: Immediate submission ($<$5 seconds to calendar servers)
    \item \texttt{ots upgrade}: 1--6 hours (awaiting Bitcoin block confirmation)
    \item \texttt{ots verify}: Instant validation (if headers available locally)
    \item Pending proofs: Cannot be verified until Bitcoin block inclusion
\end{itemize}

\subsection{CI/CD integration strategy (ADR-007):}
\begin{itemize}[nosep]
    \item Headers-only Bitcoin Core: $<$1 GB storage (vs. $>$500 GB full node)
    \item Initial header sync: \textasciitilde15 minutes (one-time cost per CI runner)
    \item Cached headers: Subsequent verifications in seconds
    \item Graceful fallback: Tests use placeholder proofs when OTS unavailable
    \item Real OTS tests: Optional via \texttt{RUN\_REAL\_OTS=1} (requires \texttt{ots} binary)
\end{itemize}

\begin{itemize}[nosep]

    \item \texttt{hypothesis}: Property-based testing for cryptographic properties
    \item \texttt{opentimestamps-client}: Optional, for real OTS stamping/verification
\end{itemize}

\textbf{Development tools:}
\begin{itemize}[nosep]
    \item \texttt{mypy}: Static type checking (100\% coverage in gateway modules)
    \item \texttt{ruff}: Fast Python linter (replaces flake8, isort, black)
    \item \texttt{bandit}: Security static analysis (ADR-009 hardening)
\end{itemize}

We use \texttt{ruff}, \texttt{mypy}, and \texttt{bandit} for linting, type checks, and security scans respectively, and track large artifacts with Git LFS; our CI experiments use the \texttt{uv} package manager for deterministic installs~\cite{ruff,mypy,bandit,git-lfs,uv_pkg}.
