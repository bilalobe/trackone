\section{Experiments}

\subsection{Determinism (M\#0--M\#3)}
We verify deterministic Merkle root computation:
\begin{itemize}[nosep]
    \item Same input facts → identical \texttt{merkle\_root} (hex)
    \item Same input facts → byte-identical \texttt{day.bin} (SHA-256)
    \item Hash-sorted leaves ensure order independence: shuffled fact files → same root
\end{itemize}

Test coverage: empty set, single fact, odd count, power-of-2 count, random filename ordering, Unicode edge cases.

\subsection{Cryptographic Correctness (M\#3)}
\textbf{Deterministic AEAD vectors:} XChaCha20-Poly1305 test vectors in \texttt{crypto\_test\_vectors.json} verified against PyNaCl bindings.

\textbf{Round-trip tests:}
\begin{itemize}[nosep]
    \item X25519: Key generation and shared secret agreement
    \item HKDF: Deterministic derivation per RFC 5869
    \item XChaCha20-Poly1305: Encrypt/decrypt round-trip, authentication failure detection
    \item Ed25519: Sign/verify signatures
\end{itemize}

\textbf{Property-based testing (Hypothesis):} TLV encoding round-trip with arbitrary inputs, robustness against truncated/malformed data.

\subsection{Security Testing (M\#3)}
Tamper detection tests validate AEAD integrity:
\begin{itemize}[nosep]
    \item \textbf{Ciphertext tampering}: Single-bit flip → tag verification fails
    \item \textbf{Tag tampering}: Modified authentication tag → rejection
    \item \textbf{AAD tampering}: Modified header fields → tag mismatch
    \item \textbf{Nonce validation}: Invalid nonce length (not 24 bytes) → rejected
    \item \textbf{Unknown device}: Frame from unprovisioned device → rejected
\end{itemize}

All security tests pass with proper error messages and logging.

\subsection{Framed Ingest Robustness (M\#1--M\#3)}
Replay window enforcement with attack scenarios:
\begin{itemize}[nosep]
    \item \textbf{Monotonic acceptance}: 10 frames with increasing \texttt{fc} (0..9) → all accepted
    \item \textbf{Duplicate rejection}: Replay same frame → rejected with logged warning
    \item \textbf{Out-of-window rejection}: Jump \texttt{fc} by +100 beyond window (64) → rejected
    \item \textbf{Reordering tolerance}: Frames within window but out of order → accepted
    \item \textbf{Restart persistence}: Device table persisted → duplicates rejected after restart
\end{itemize}

\subsection{End-to-End Pipeline (M\#3)}
Complete workflow validated by \texttt{test\_end\_to\_end\_pipeline}:
\begin{enumerate}[nosep]
    \item \texttt{pod\_sim --framed} generates encrypted frames
    \item \texttt{frame\_verifier} decrypts with XChaCha20-Poly1305, validates tags
    \item \texttt{merkle\_batcher} builds Merkle tree, validates schemas
    \item \texttt{ots\_anchor} creates OTS proof (placeholder or real)
    \item \texttt{verify\_cli} recomputes root, verifies match and OTS timestamp
\end{enumerate}

Exit code 0 confirms entire chain is cryptographically verifiable.
